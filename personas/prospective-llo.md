# Persona — Prospective LLO

A Local Implementation Organization considering whether to bid on this opp. Reads the demo to gauge feasibility, scope, supervision burden, and whether the FLW-facing tooling is something their existing field staff can absorb.

## Priorities

- **Operational realism.** What does a typical FLW day look like? How many visits/week, what's the supervision rhythm, how does data flow back to the LLO?
- **Quality signals they can act on.** If a CHW underperforms, can the LLO see it early? Are flags actionable (named + dated) or noise (just colors)?
- **Equipment + skill match.** Does the survey assume infrastructure (smartphones, GPS accuracy, mobile data) and skill (literacy, language, comfort with media capture) that the LLO's roster actually has?
- **Total cost of supervision.** Is the LLO weekly review they're shown a 30-min affair or a 4-hour one?

## Language

- "FLW" / "field worker" / "CHW" — match the term the program uses; the LLO is on the field side, not the funder side.
- Concrete numbers, not adjectives. "92% completeness" beats "high quality."
- Cycle / rotation language: "week one," "by end of month," "before close-out."

## Dashboards of interest

- **LLO weekly review** (when it lands in Stage 3) — they want to see what THEIR review will look like.
- **Per-FLW performance pages** — the rockstar / steady / struggling / new-hire breakdown is exactly the lens they manage by.
- **Visit map / list view** — geographic coverage, refusals, repeat-vendor dedup.
- **Coaching task drawer** (Stage 3) — what does following up with an underperforming FLW look like?

## Turn-offs

- Generic dashboard demos that could be any program — they want to see THIS program's data.
- Donor-pitch language ("transformational," "scalable") — sounds like they're being sold to.
- Tools that require a tech-savvy supervisor; LLOs value tools their existing supervisors can drive after one session.
- Raw JSON / IDs visible — looks like the prototype isn't done.
- Empty states / "no data yet" — they want to see the system when it's working, not when it's blank.

## What lands

- Named FLWs with archetype labels and specific weekly numbers ("Asha did 38 visits, 92% completeness, mentored 2 peers").
- Anomalies surfaced with context, not just severity flags ("Dinesh's prices look 2-3× the market median in Week 3 — possible currency confusion").
- A path from anomaly → coaching task → re-measurement showing the loop closes.
- "What we'd do if we awarded you this opp" framing.
